Checking schema conflicts¶

Column Names¶

The column names in the recent disaster dataset (S2) is incorrectly labeled. It can be seen that according to http://www.emdat.be/classification, which was where the dataset was recieved from, the column that cotains information on the name of the country is meant to be country name, and the column next to it is meant to be disaster group, hence the column name is shifted one column to the right after Country. The semicolon between country iso and country name was not meant to be there.

Some findings:

  • Occurence is the number of disaster occurence, hence should be an int not an object, and realistically a disaster cannot occur 30 millions times, hence this column should be Total death instead.
  • Theoretically the number of people affected should be greater than the number of people injured
  • By shifting the column one to the right, unnamed: 16 becomes international support, and international support becomes total damage

def semantic_mapping(S1, S2):
    S1colname = []
    S2colname = []
    for i in S1.columns:
        for j in S2.columns:
            if ((str(i) == str(j)) & (S1[str(i)].dtype == S2[str(j)].dtype)): #data type matchin, and column name match
                S1colname.append(i)
                S2colname.append(j)
    similarities = pd.DataFrame({'S1 Column': S1colname, 'S2 Column': S2colname}) #creates dataframe for matching pair
    return(similarities)
